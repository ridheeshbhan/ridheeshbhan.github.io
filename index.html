Sales and Revenue - Power BI

One of our clients wanted to get a dashboard that connects with Sage(ERP) and show their Sales, Purchases, Stock, Nominal Ledger and Net Revenue. Sage had a connection with Power BI using AAD.

They requested for some drill downs on NL (Code -> Department -> Cost Centre) and some other filters/drill downs that would dynamically refresh the data. We had to clean and transform the data in Power Query. It was important to keep data that is invoiced and remove Sales/Purchases that were not. The connection was unable to get details on invoiced data so we had to manually create a relation using helper columns.

Their companies data was then compared with similar organizations to give them a real picture of the business. This data was collected by the client in an excel file.

Sales Dashboard - Excel

The client wanted to get a Pivot Table with his Sales data based on Sales Rep -> Sales Region -> Stock details at the minimum cost possible. He had a temp solution based on thousands of formulas that took ages for him to filter the data and get the right results. His objective was to get the data and email it to individuals every Monday.

We created a Pivot table in Excel that worked on a SQL view created for partial cleaning of data. We did not clean the data fully in SQL because our client had further plans to add KPIs and Visuals. We added buttons to filter the Pivot Table by current month, current year, last 3 years, etc. We added another button to send an email to Sales Rep with only their data, the button had a macro that would filter data for a Rep, create a new file with the data, store that data on our client's server backup and send that file in email. This was all hardcoded because our company was charging them based on licenses, we added sales rep's email ids in the macro itself.

The project had further developments as our client was surprised with how robust and powerful excel could be.
